Seccion de preguntas sobre HTTP

  • Sobre que protocolo binario está montado el protocolo HTTP?

    • El protocolo HTTP esta montado sobre el protocolo de comunicaciones TCP. TCP establece una conexion entre el cliente y el servidor mediante un saludo de 3 vias (solicitud, aceptacion y confirmacion de la conexion). Esta conexion se cierra de manera ordenada una vez que finaliza la transmision de datos.

  • Cuáles son los clientes HTTP y los servidores http mas utilizados ?

    • Los clientes HTTP mas usados son los navegadores de las computadoras que usamos habitualmente (Chrome, Mozilla). Mientras que los servidores HTTP mas utilizados son Apache y Microsoft Internet Information Services (IIS), entre otros.

  • Que verbos admite un comando en el requerimiento HTTP?

    • Cada comando de requerimiento HTTP lleva en su linea inicial un verbo que le indica al servidor la naturaleza del mismo:

    • GET: Se utiliza para solicitar datos de un recurso específico. El servidor responde proporcionando los datos solicitados, si están disponibles.
    • POST: Se utiliza para enviar datos al servidor para su procesamiento. Este verbo es comúnmente utilizado cuando se envían formularios en línea o se realizan operaciones que modifican datos en el servidor.
    • PUT: Se utiliza para crear recursos en el servidor.
    • DELETE: Se utiliza para eliminar un recurso en el servidor.
    • PATCH: Se utiliza para realizar modificaciones parciales a un recurso. Esto significa que solo se actualizan los campos especificados en la solicitud, en lugar de reemplazar todo el recurso.

  • Que contenido lleva el body de un requerimiento HTTP?

    • El body de un requerimiento HTTP puede contener datos de cualquier formato, por ejemplo, los datos de un formulario que esta enviando el cliente al servidor

  • Que diferencia existe entre un URL y un URI?

    • Mientras que un URL especifica la ubicación precisa de un recurso en la web, un URI puede ser más general y puede identificar cualquier tipo de recurso en una red.

  • Como se almacena información relacionada con las respuestas HTTP?

    • La informacion de las respuestas HTTP se almacenan mediante cookies. Las cookies son ampliamente utilizadas en la web para una variedad de propósitos, como la autenticación de usuarios, el seguimiento de la actividad del usuario, la personalización del contenido y la publicidad dirigida.

  • Que significa Virtual hosting?

    • Se denomina Virtual Hosting a la práctica de alojar múltiples sitios web en un mismo servidor físico. En lugar de asignar un servidor dedicado a cada sitio web, se utilizan técnicas de configuración para que un solo servidor pueda servir varios dominios de manera simultánea.

  • Porque el HTTP se considera STATE LESS?

    • HTTP se considera "stateless" (sin estado) porque cada solicitud HTTP que realiza un cliente al servidor es independiente y no guarda ninguna información sobre las solicitudes anteriores. Esto significa que el servidor no mantiene ningún conocimiento del estado anterior de las interacciones con el cliente.

  • Como va a mejorar la velocidad de la WEB en poco tiempo?

    • La velocidad de la web mejora gracias a que el protocolo HTTP/2, a diferencia de HTTP/1.1, trabaja con paquetes binarios y permite la multiplexacion de solicitudes y respuestas en una misma conexion TCP.

  • Como va a mejorar el comportamiento de una aplicación WEB en ambientes ruidosos o de débil conexión?

    • Las conexiones en ambientes de debil conexion se mejoraran gracias al protocolo HTTP/3, que se basa en el protocolo QUIC (desarrollado por Google), que utiliza UDP en lugar de TCP para el transporte de datos. UDP es menos susceptible a problemas de congestión y pérdida de paquetes en comparación con TCP, lo que puede mejorar el rendimiento en entornos de red inestables.
